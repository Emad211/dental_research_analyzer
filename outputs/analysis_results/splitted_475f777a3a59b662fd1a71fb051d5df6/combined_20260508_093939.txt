## TASK 2 — Statistical analysis plan (SAP)

### 0) General principles (apply to all analyses)
- **Unit of analysis:** instrument.
- **Analysis sets:**
  - Primary: all tested instruments with valid fracture time; include censored observations (no fracture by 600 s).
- **Descriptives:**
  - Continuous variables: mean/SD if ~normal; otherwise median/IQR; always report **n**, **# censored**.
  - Graphs: box/violin plots for NCF/time; Kaplan–Meier curves for time-to-fracture by system.
- **Multiplicity:** if comparing multiple systems pairwise, control family-wise error (Tukey/Holm) or FDR (Benjamini–Hochberg) depending on objective.
- **Assumption checks:** normality (QQ plot/Shapiro–Wilk), homoscedasticity (Levene/Brown–Forsythe), outliers/influence.
- **Key design issue:** Because tests stop at 600 s, **time-to-fracture and NCF are right-censored** for instruments not failing; any method treating those as exact 600 s (or excluding them) can bias estimates and p-values. Survival methods are strongly preferred if any censoring occurred.

## CATEGORY A — MANDATORY tests (analysis incomplete without them)

### A1. Kaplan–Meier survival analysis + log-rank test
- **Test name:** Kaplan–Meier estimator; **log-rank test** for group comparison.
- **Purpose:** Compare **time-to-fracture** distributions across instrument systems while properly handling **right-censoring at 600 s**.
- **Why appropriate (≥2):**
  1. Explicitly accommodates **censored observations** (no fracture within test window).
  2. Compares entire survival curves, not just means, capturing distributional differences in failure times.
- **Why may not be appropriate (≥1):**
  - If **no censoring** and all instruments fracture well before 600 s, survival methods are not necessary (though still valid).
- **When NOT to use:**
  - If outcome is not time-to-event (e.g., single continuous measurement without censoring) or if censoring is not present/possible by design.
- **Alternative if assumptions violated:**
  - If hazards cross heavily (non-proportionality), prefer **weighted log-rank tests** (e.g., Fleming–Harrington) or model-based approaches (A2/A3).

### A2. Cox proportional hazards regression (multivariable survival model)
- **Test name:** Cox PH model.
- **Purpose:** Estimate **hazard ratios** for fracture by instrument system (and optionally adjust for test settings such as rpm/torque if treated as covariates, though they are system-determined).
- **Why appropriate (≥2):**
  1. Handles **censoring** and yields interpretable **effect sizes** (HR with 95% CI).
  2. Allows **covariate adjustment** and formal assessment of factors (system; possibly rpm) if needed.
- **Why may not be appropriate (≥1):**
  - The **proportional hazards assumption** may be violated (different fatigue behavior over time).
- **When NOT to use:**
  - When PH assumption is badly violated and cannot be remedied (e.g., time-varying effects) and sample size is too small for stratification/time-varying terms.
- **Alternative if assumptions violated:**
  - **Stratified Cox model**, **Cox with time-varying coefficients**, or **accelerated failure time (AFT) model** (A3).

### A3. Accelerated failure time (AFT) model (e.g., log-normal or Weibull)
- **Test name:** Parametric survival regression (AFT).
- **Purpose:** Quantify how instrument system **accelerates/decelerates** time-to-fracture; often more interpretable in fatigue testing (“time ratio”).
- **Why appropriate (≥2):**
  1. Handles **right-censoring** and can outperform Cox when a parametric distribution fits well.
  2. Provides **time ratios** (e.g., “System A lasts 1.6× longer than System B”).
- **Why may not be appropriate (≥1):**
  - Misspecification of the survival-time distribution can bias results.
- **When NOT to use:**
  - When no plausible parametric form fits (poor goodness-of-fit) or sample size is too small to discriminate distributions.
- **Alternative if assumptions violated:**
  - Revert to **Cox PH** (A2) or nonparametric comparisons (A1).

## CATEGORY B — RECOMMENDED tests (improve rigor/interpretability)

### B1. Global comparison of NCF across systems using ANOVA (or Welch ANOVA) **only if no censoring**
- **Test name:** One-way ANOVA (or **Welch ANOVA**).
- **Purpose:** Compare mean **NCF** among instrument systems when NCF is fully observed.
- **Why appropriate (≥2):**
  1. NCF is continuous and group membership is categorical.
  2. Efficient for detecting mean differences with balanced group sizes (n≈20/group).
- **Why may not be appropriate (≥1):**
  - If any censoring exists, NCF is **not fully observed**, and treating censored values as fixed (e.g., NCF at 600 s) biases comparisons.
- **When NOT to use:**
  - When tests are censored or assumptions of normality/variance are materially violated without remedy.
- **Alternative if assumptions violated:**
  - If no censoring but heteroscedasticity: **Welch ANOVA** + Games–Howell post hoc.
  - If non-normal: **Kruskal–Wallis** (B2).
  - If censoring: survival methods (A1–A3) should replace this as primary.

### B2. Kruskal–Wallis test (+ Dunn post hoc) for NCF/time when non-normal (and uncensored)
- **Test name:** Kruskal–Wallis; Dunn’s test with multiplicity correction.
- **Purpose:** Compare central tendency across systems without assuming normality.
- **Why appropriate (≥2):**
  1. Robust to non-normal distributions common in fatigue data.
  2. Works with small-to-moderate n/group.
- **Why may not be appropriate (≥1):**
  - Still assumes **independent samples** and does not address **censoring**.
- **When NOT to use:**
  - When censoring is present or when interest is explicitly in mean differences (it tests stochastic dominance/median-type differences).
- **Alternative if assumptions violated:**
  - With censoring: **log-rank/Cox/AFT** (A1–A3).

### B3. Post hoc multiple comparisons (pairwise) with multiplicity control
- **Test name:** Tukey HSD (after ANOVA), Games–Howell (after Welch), or Holm-adjusted pairwise tests; for survival: pairwise log-rank with Holm/FDR.
- **Purpose:** Identify which instrument pairs differ.
- **Why appropriate (≥2):**
  1. Multiple systems → many pairwise contrasts; multiplicity control prevents inflated Type I error.
  2. Provides actionable comparisons (which instruments differ).
- **Why may not be appropriate (≥1):**
  - Small sample sizes can yield low power after correction.
- **When NOT to use:**
  - If only an overall/global hypothesis is of interest or if post hoc testing was not pre-specified and risks overinterpretation.
- **Alternative if assumptions violated:**
  - Use robust/nonparametric pairwise methods consistent with the main model (e.g., Dunn, pairwise log-rank).

### B4. Analysis of fragment length across systems
- **Test name:** One-way ANOVA/Welch or Kruskal–Wallis (depending on assumptions).
- **Purpose:** Check whether **fracture location** differs by instrument system (quality control; may reflect canal curvature engagement).
- **Why appropriate (≥2):**
  1. Fragment length is continuous and may indicate whether failures occurred at comparable canal positions.
  2. Helps confirm testing comparability across groups (a methodological validity check).
- **Why may not be appropriate (≥1):**
  - If fragment length is missing for censored cases (no fracture), analysis is conditional on failure and may be biased.
- **When NOT to use:**
  - If many instruments are censored (no fracture), making fragment-length comparison unrepresentative.
- **Alternative if assumptions violated:**
  - Robust methods (Kruskal–Wallis) or model fragment length only among fractured instruments with transparent reporting.

### B5. Assumption/diagnostic tests (as part of QA)
- **Tests:** Shapiro–Wilk (normality), Levene/Brown–Forsythe (equal variances), PH assumption tests (Schoenfeld residuals) for Cox.
- **Purpose:** Validate model choice and report robustness.
- **Why appropriate (≥2):**
  1. Fatigue data often skewed; variance often differs by system.
  2. Cox models require PH; checking is essential for correct inference.
- **Why may not be appropriate (≥1):**
  - With small n, assumption tests can be underpowered/overly sensitive; should be complemented by plots.
- **When NOT to use:**
  - Never as the sole basis for decisions; avoid automatic “test then choose” without scientific judgment.
- **Alternative if assumptions violated:**
  - Switch to robust methods (Welch, nonparametric) or alternative survival models (AFT, time-varying Cox).

## CATEGORY C — OPTIONAL analyses (supplementary insight)

### C1. Effect size reporting
- **Measures:** For ANOVA-type: η²/ω²; for pairwise: Cohen’s d or Cliff’s delta; for survival: HRs and/or time ratios; restricted mean survival time (RMST) differences.
- **Purpose:** Quantify magnitude beyond p-values.
- **Why appropriate (≥2):**
  1. Enhances interpretability and clinical/material relevance.
  2. Allows comparison to prior studies and supports meta-analysis.
- **Why may not be appropriate (≥1):**
  - Some effect sizes are distribution/scale dependent; RMST depends on truncation time (600 s).
- **When NOT to use:**
  - Avoid Cohen’s d on heavily skewed outcomes; avoid mean-based measures when censoring is substantial.
- **Alternative if assumptions violated:**
  - Use **RMST** (survival) or **Cliff’s delta** (nonparametric).

### C2. RMST analysis (restricted mean survival time) up to 600 s
- **Test name:** RMST difference/regression.
- **Purpose:** Compare average event-free time **within the 600 s window**.
- **Why appropriate (≥2):**
  1. Works well with censoring and does **not require PH assumption**.
  2. Directly aligned with the experimental truncation at 600 s.
- **Why may not be appropriate (≥1):**
  - Interpretation is limited to the restriction horizon (600 s).
- **When NOT to use:**
  - If the truncation time is arbitrary and far from most events (though here it is design-defined).
- **Alternative if assumptions violated:**
  - Cox/AFT remain alternatives; RMST is itself robust.

### C3. Sensitivity analysis using time-to-fracture vs NCF
- **Approach:** Repeat primary comparisons using both **time-to-fracture** and **NCF**, clarifying interpretation given differing rpm settings across systems.
- **Purpose:** Confirm conclusions are not an artifact of rpm conversion.
- **Why appropriate (≥2):**
  1. rpm differs by system; NCF adjusts for rpm, time does not.
  2. Demonstrates robustness to outcome scale.
- **Why may not be appropriate (≥1):**
  - If rpm is part of the “system package,” adjusting may remove a meaningful component of performance.
- **When NOT to use:**
  - If the scientific question is strictly about clinical working time (seconds) rather than cycles.
- **Alternative if assumptions violated:**
  - Model both outcomes with appropriate methods (survival for time; survival on cycles if censoring can be translated).

### C4. Reliability of video-derived timing (if multiple raters)
- **Test name:** Intraclass correlation coefficient (ICC) or Bland–Altman.
- **Purpose:** Quantify measurement repeatability of start/end time extraction.
- **Why appropriate (≥2):**
  1. Timing is derived from video—potential observer variability.
  2. Supports measurement validity.
- **Why may not be appropriate (≥1):**
  - Not needed if times are extracted automatically or by a single rater without repeats.
- **When NOT to use:**
  - When no replicate measurements exist.
- **Alternative if assumptions violated:**
  - Use weighted kappa for categorized times (not ideal) or repeat measurement protocol.

## TASK 3 — Inappropriate tests (should NOT be used)

1. **Chi-square tests / Fisher’s exact test as primary analysis**
   - Outcome is continuous time/cycles, not categorical counts; would require arbitrary dichotomization (e.g., “fractured by 600 s yes/no”), losing information.

2. **Multiple independent t-tests without multiplicity correction**
   - Many systems → inflated Type I error; violates family-wise error control.

3. **Paired t-test / repeated-measures ANOVA**
   - Instruments are independent; no within-instrument repeated outcome was described.

4. **Simple linear regression on time/NCF ignoring censoring**
   - If any instruments do not fracture by 600 s, treating them as observed (or deleting them) biases estimates and p-values.

5. **Pearson correlation between rpm and NCF across instruments**
   - rpm is set by manufacturer and essentially collinear with “system”; correlation would be confounded and not meaningful.

6. **MANOVA as primary**
   - Primary scientific endpoint is fatigue resistance; additional variables (fragment length) are ancillary and may be missing for censored cases, violating MANOVA requirements.

## TASK 3: INAPPROPRIATE TESTS AND WHY

Below are statistical tests/methods that should NOT be used for this study, with rationale.

1. **Independent-samples t-test for all pairwise comparisons without correction**
   - **Why inappropriate:**  
     - There are 6 groups; pairwise t-tests (15 comparisons) without multiplicity correction inflate type I error substantially.  
   - **Better alternative:** ANOVA + appropriate post hoc procedure.

2. **Paired t-test or repeated-measures ANOVA**
   - **Why inappropriate:**  
     - Each instrument is measured once and belongs to a single group; there are no repeated measurements on the same unit.  
   - **Better alternative:** One-way ANOVA (independent groups).

3. **Chi-square test for comparing NCF or time to fracture**
   - **Why inappropriate:**  
     - NCF and time to fracture are continuous; dichotomizing them (e.g., high vs low fatigue resistance) wastes information and reduces power.  
   - **Better alternative:** ANOVA, Kruskal–Wallis, or survival methods on continuous data.

4. **Simple linear regression of NCF on rpm alone to infer instrument differences**
   - **Why inappropriate:**  
     - RPM is fixed by design for each instrument; regression of NCF on rpm confounds instrumental and design differences and does not answer the question about instrument/alloy/core mass.  
   - **Better alternative:** Group-based comparisons with rpm as covariate in a multi-variable model if needed.

5. **Correlation between instrument group (categorical) and NCF using Pearson correlation**
   - **Why inappropriate:**  
     - Pearson correlation requires numeric variables; coding categories as arbitrary numbers (1–6) misrepresents relationships.  
   - **Better alternative:** ANOVA or regression with dummy variables.

6. **Student’s t-test to compare more than two groups simultaneously**
   - **Why inappropriate:**  
     - t-test is limited to two groups; multiple t-tests across several groups leads to inflated type I error and ignores overall group structure.  
   - **Better alternative:** One-way ANOVA or Kruskal–Wallis.

7. **Parametric methods ignoring censoring (e.g., ANOVA on time to fracture when many tests end at 600 s without proper handling)**
   - **Why inappropriate:**  
     - Censored observations are not true times; treating them as equal to 600 s biases estimates and violates assumptions.

8. **Logistic regression modelling “fracture vs no fracture by 600 s” as the primary outcome**
   - **Why inappropriate:**  
     - There is rich information on the time to fracture; collapsing to a binary indicator loses resolution and power.  
   - **Better alternative:** Survival analysis; NCF/time to fracture as continuous outcome.

9. **MANOVA on NCF and time to fracture simultaneously**
   - **Why inappropriate:**  
     - Time to fracture and NCF are linearly related given rpm; they are not independent multivariate outcomes, so MANOVA adds no value and can violate assumptions.  
   - **Better alternative:** Analyze one primary outcome (NCF or time) and, if desired, the other as supportive.

    2) The Weibull modulus (shape parameter) provides insight into the reliability and predictability of the instrument's failure, which means more clinically than just "average cycles to failure."
*   **Why it may not be appropriate:** It requires a specific data distribution; if the failure mechanism is mixed (e.g., both torsional and cyclic), a single Weibull distribution may not fit well.
*   **When this test should NOT be used:** For analyzing the fragment length data, as this is not a time-to-failure metric.
*   **Alternative if assumptions are violated:** Lognormal distribution survival analysis.

#### CATEGORY C – OPTIONAL TESTS
*Tests that provide supplementary insights.*

**1. Pearson Correlation Coefficient**
*   **Purpose:** To evaluate the linear relationship between Fragment Length and NCF.
*   **Why appropriate:** 
    1) It can reveal if instruments that break further up the shaft (longer fragments) survive longer or shorter. 
    2) It provides insight into the biomechanics of the 60° curvature stress point.
*   **Why it may not be appropriate:** It assumes a linear relationship; the relationship between fracture location and fatigue life might be non-linear.
*   **When this test should NOT be used:** If the data for either variable is heavily skewed or contains extreme outliers.
*   **Alternative if assumptions are violated:** Spearman’s Rank Correlation.

### TASK 3: INAPPROPRIATE TESTS

*   **Independent Samples t-test:** Should NOT be used because there are 7 groups. Running multiple t-tests (e.g., Group 1 vs 2, Group 1 vs 3) will drastically inflate the Type I error rate (false positives).
*   **Paired t-test / Repeated Measures ANOVA:** Should NOT be used because the instruments are independent. An instrument is tested until it breaks; it cannot be tested again. There are no repeated measures on the same subject.
*   **Chi-Square Test of Independence (for NCF):** Should NOT be used because NCF is a continuous numerical variable, not categorical. (It *could* be used for fracture mode, but since the authors are just verifying cyclic fatigue to exclude torsional fracture, the variance will likely be zero).
*   **Multivariate Analysis of Variance (MANOVA):** While there are two dependent variables (NCF and fragment length), they measure entirely different physical phenomena. Combining them into a single synthetic variable via MANOVA would complicate clinical interpretation without adding value.
