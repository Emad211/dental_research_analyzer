## CATEGORY A – MANDATORY tests

### 1) McNemar test (paired binary outcome)
- **Purpose:** Test whether the **probability of extrusion differs** between test and control **within the same tooth**.
- **Why appropriate (≥2 reasons):**
  1. Outcome is **binary** (E vs NE).
  2. Design creates **matched pairs** (each tooth contributes one test canal and one control canal), violating independence required for a standard chi-square.
- **Why it may not be appropriate (≥1 reason):**
  - If pairing is not truly 1:1 within tooth (e.g., some teeth contribute only one canal or allocation not paired), McNemar is not valid.
- **When this test should NOT be used:**
  - When observations are **independent** (parallel-group randomization by tooth/child) or when there are **>2 correlated observations per cluster** without clear pairing.
- **Alternative if assumptions are violated:**
  - **GEE logistic regression** with cluster = tooth (or child) for correlated binary outcomes; or **mixed-effects logistic regression**.

### 2) Effect size estimation with 95% CI (paired)
(Conceptually mandatory even if not a “test”.)
- **Purpose:** Quantify magnitude: e.g., **paired risk difference** (Δ risk of extrusion) and/or **matched odds ratio**.
- **Why appropriate:**
  1. RCT interpretation requires **estimation**, not only p-values.
  2. Provides clinical relevance (e.g., absolute reduction in extrusion).
- **Why it may not be appropriate:**
  - With **very low event counts**, CIs may be wide/unstable.
- **When it should NOT be used:**
  - Never omit estimation; but avoid asymptotic CI formulas if sparse.
- **Alternative if assumptions violated:**
  - **Exact** CI methods for matched pairs; or **bootstrap by tooth**.

## CATEGORY B – RECOMMENDED tests/models

### 3) Conditional logistic regression (matched-pair logistic model)
- **Purpose:** Estimate **matched odds ratio** for extrusion (test vs control) while respecting within-tooth matching; optionally adjust for canal-level covariates (e.g., mesial/distal) if recorded.
- **Why appropriate:**
  1. Directly models **paired binary outcomes** consistent with split-tooth randomization.
  2. Can incorporate **covariates** (canal type, degree of resorption if measured) and yields adjusted effect estimates with CIs.
- **Why it may not be appropriate:**
  - Sparse outcomes can cause **separation** or unstable estimates.
- **When it should NOT be used:**
  - If there is **no matching** or matching factors are incorrectly specified; or if only marginal inference is desired and pairing is complex.
- **Alternative if assumptions are violated:**
  - **Exact conditional logistic regression** (if sparse) or **GEE logistic** with robust SEs.

### 4) GEE logistic regression (population-averaged)
- **Purpose:** Estimate treatment effect on extrusion with **robust standard errors** accounting for within-tooth correlation.
- **Why appropriate:**
  1. Handles **correlated binary** outcomes without requiring correct random-effects distribution.
  2. Provides **marginal (population-averaged)** effect, often preferred for clinical interpretation.
- **Why it may not be appropriate:**
  - With small numbers of clusters (here ~47 teeth), sandwich SEs can be **biased downward**.
- **When it should NOT be used:**
  - Very small cluster count without small-sample corrections; or if interest is specifically subject/tooth-specific effects.
- **Alternative if assumptions are violated:**
  - **Mixed-effects logistic regression**; or GEE with **small-sample correction** (e.g., Fay–Graubard / Kauermann–Carroll corrections, depending on software).

### 5) Fisher’s exact test / chi-square test (as a *sensitivity* analysis only)
- **Purpose:** Compare extrusion proportions between groups **ignoring pairing** (not primary).
- **Why appropriate:**
  1. Provides a simple cross-check of direction/magnitude.
  2. Fisher’s exact can be valid with **small counts** (but still assumes independence).
- **Why it may not be appropriate:**
  - **Violates independence** because canals within a tooth are paired; p-values can be misleading.
- **When it should NOT be used:**
  - As the main inferential test for this split-tooth design.
- **Alternative if assumptions are violated:**
  - Use **McNemar** / **conditional logistic** instead (i.e., revert to paired methods).
